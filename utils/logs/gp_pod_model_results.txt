Takeaways
---------
- Both runs completed correctly end-to-end on the real (full) local dataset.
- Full default config (44 modes / 99% variance) finished in under a minute
  total on CPU -- confirms this model does NOT need the cluster/GPU. The
  whole fit operates on compact (n_train x n_modes) matrices, nothing like
  the full-field tensors that make the KLw-loss MLP / pointwise XGBoost
  scripts in this repo need a GPU.
- Predictive std tracks actual difficulty sensibly: phase 2 (harder, worse
  R2/KLw) gets a higher self-reported std (0.057) than phase 1 (0.025) --
  the Bayesian uncertainty is doing something meaningful, not just noise.
- Phase 1 R2 ~0.97, phase 2 R2 ~0.86; phase 2's worse KLw (mean 9.4 vs 2.7)
  is concentrated in wing/pylon (highest weighted components), consistent
  with phase 2 being the harder/more OOD-like conditions.

Artifacts from these runs were written to the scratchpad, not committed here
(model.joblib, y_pred*.npy, std_pred*.npy) -- rerun the script to regenerate
them if needed; this file is just the numbers for reference.
